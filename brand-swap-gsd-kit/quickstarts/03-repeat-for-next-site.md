# Quickstart 03 — Repeat For The Next Site

Use this to rinse-and-repeat across the rest of your scaffolded sites.

## Repeat loop
1. Drop `brand-swap-gsd-kit/` into the new repo.
2. Copy `workflows/brand-swap-init-gsd.json` into `.vscode/workflows/`.
3. Copy the templates into:
   - `.plan/SOURCE_OF_TRUTH.json`
   - `docs/plans/<date>-swap-features.json`
4. Run the Init phase:
   - `app-spec-agent`
   - `Init-agent`
5. Fill the manifest with the site-specific:
   - typography map
   - logo size and placement
   - color roles
   - media requirements
   - copy blocks
   - locked systems
6. Stay in planning-only mode until assets exist.
7. When ready, hand the repo to GSD for planning and execution.

## Success criteria
For every site, you should end up with:
- one manifest
- one source-of-truth pointer
- one clear asset intake list
- one reusable execution path
